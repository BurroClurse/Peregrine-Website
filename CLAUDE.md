# CLAUDE.md ‚Äî Peregrine marketing site (peregrinedryfire.com)

Standalone HTML/CSS/JS landing page for the Peregrine dry-fire iOS app.
No build step. The app repo lives at `~/Developer/Peregrine-iOS` (drift-zone
copy in `Test4-LaserDotDetector/Detection/DriftAnalyzer.swift` is the source
of truth for the drift wheel's text).

## Run & test

```bash
python3 serve.py                        # http://localhost:8099 (range-capable;
                                        # plain http.server breaks Safari video)
./publish.sh                            # refresh public/ for Netlify
node tests/editor-regression.test.js    # regression checks (plain script ‚Äî do
                                        # NOT use `node --test`)
```

serve.py sends `Cache-Control: no-cache` for html/css/js; images/video stay
cacheable (bust them with `fetch(url, {cache:'reload'})` when re-encoded).

## Files

- `public/` ‚Äî generated/ignored output and the only folder Netlify publishes,
  configured in `netlify.toml`. Netlify runs `./publish.sh` before publishing.
  `public/index.html`, `public/styles.css`, `public/script.js`, and
  `public/assets/` are generated by the script.
- `index.editor.html` ‚Äî authoring page with the Customize/Edit controls. This
  is committed to GitHub but not served by Netlify.
- `index.html` ‚Äî clean root source snapshot; Netlify ignores this copy.
- `styles.css` ‚Äî source design tokens at top; drift wheel styles under `dw-`.
- `script.js` ‚Äî source runtime (dependency-free; must survive editor Export).
  Drift wheel, carousels, count-up, laser ping, video handling.
- `editor.js` / `editor.css` ‚Äî the dev-only Customize panel. Images live in
  IndexedDB (`peregrineEditorAssets`, `pe-asset:` refs); Export writes real
  files to `assets/edited/` via showDirectoryPicker. Run `./publish.sh` to
  strip editor refs + `#peDock` + `#pe-saved-state` into `public/`.
- `docs/specs/`, `docs/plans/` ‚Äî design docs.

## Git

If git *hangs*, iCloud evicted files: `brctl download ~/Desktop/Peregrine-Website`.

## Gotchas

- Device frames: children size chrome in `cqw` (container queries), but the
  frame's own box must stay px ‚Äî an element's cqw can't reference itself.
- Editor Layout selection binds `pointerdown` on `[data-pe-move]`, not click.
- Claude-in-Chrome MCP tabs are `document.hidden`: rAF/IntersectionObserver
  are paused and `video.play()` awaits hang ‚Äî verify animations in headless
  puppeteer; `navigate` to the already-open URL does not reload (use
  `location.reload()` via JS).
